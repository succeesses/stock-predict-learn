# 赵鑫海的微调预测方案

## 方案概述

对于股价每日更新滚动预测场景，推荐使用 **「定期微调 + 每日滑动预测」** 方案。该方案在预测效果和计算成本之间取得了良好平衡。

## 整体流程

### 第一步：初始化 - 一次性微调模型

使用目标股票迄今为止的**全部历史数据**进行一次完整微调：

1. 准备CSV数据文件，包含列：
   ```
   timestamps,open,high,low,close,volume,amount
   ```

2. 修改配置文件 `finetune_csv/configs/your_config.yaml`：
   ```yaml
   data:
     data_path: "/path/to/your/stock_data.csv"
     lookback_window: 400      # 使用最近多少根K线作为历史上下文
     predict_window: 1         # 只预测明天（下一个交易日）
     max_context: 512
   ```

3. 运行训练：
   ```bash
   python train_sequential.py --config configs/your_config.yaml
   ```

4. 保存微调后的模型，路径会自动生成在配置的 `base_save_path` 下。

---

### 第二步：每日预测流程（无需训练，快速推理）

每日收盘后执行以下步骤，预测下一个交易日：

```python
import pandas as pd
from model import Kronos, KronosTokenizer, KronosPredictor

# 1. 更新数据：下载今天最新数据，追加到CSV文件
df = pd.read_csv("your_stock_data.csv")
# 假设 today_new_data 是你刚获取的今天K线数据，格式正确
df = pd.concat([df, today_new_data], ignore_index=True)
df.to_csv("your_stock_data.csv", index=False)

# 2. 加载已经微调好的模型（不需要重新训练）
tokenizer = KronosTokenizer.from_pretrained("./path/to/finetuned/tokenizer/best_model")
model = Kronos.from_pretrained("./path/to/finetuned/basemodel/best_model")
predictor = KronosPredictor(model, tokenizer, max_context=512)

# 3. 截取最新的 lookback 根K线作为输入
lookback = 400  # 使用最近400天历史数据（根据你的配置调整）
x_df = df.tail(lookback)[['open', 'high', 'low', 'close', 'volume', 'amount']]
x_timestamp = df.tail(lookback)['timestamps']

# 4. 构造明天的时间戳（需要确认明天是否交易日）
tomorrow_date = next_trading_day  # 你的代码计算下一个交易日
y_timestamp = pd.Series([pd.to_datetime(tomorrow_date)])

# 5. 预测明天
pred_df = predictor.predict(
    df=x_df,
    x_timestamp=x_timestamp,
    y_timestamp=y_timestamp,
    pred_len=1,
    T=1.0,
    top_p=0.9,
    sample_count=1
)

# 获取预测结果
tomorrow_open = pred_df.iloc[0]['open']
tomorrow_high = pred_df.iloc[0]['high']
tomorrow_low = pred_df.iloc[0]['low']
tomorrow_close = pred_df.iloc[0]['close']
tomorrow_volume = pred_df.iloc[0]['volume']

print(f"明天预测: 开盘={tomorrow_open:.2f}, 最高={tomorrow_high:.2f}, 最低={tomorrow_low:.2f}, 收盘={tomorrow_close:.2f}")
```

### 第三步：定期更新模型

**不需要每日微调**，只需要定期用最新数据重新微调即可：

| 数据频率 | 推荐微调周期 | 理由 |
|---------|-------------|------|
| **日线数据** | 每周/每两周 | 每日新增数据对整体分布影响小，每周更新足够跟上市场变化 |
| **5分钟线** | 每1-3个月 | 高频数据存量大，新增数据占比小，可以更长周期微调一次 |

重新微调命令和初始化一样：
```bash
python train_sequential.py --config configs/your_config.yaml
```

---

## 方案优势

1. **计算成本低**：每日只需要推理，不需要训练，几秒钟就能出结果
2. **效果足够好**：一天新增的数据对模型参数影响极小，参数短期不会明显漂移
3. **模型与时俱进**：定期微调让模型参数能够跟上市场结构变化
4. **实现简单**：不需要复杂的增量训练逻辑，代码维护容易

---

## 为什么不推荐「每日微调」

| 问题 | 说明 |
|------|------|
| **计算量大** | 每天都要跑训练，浪费时间和GPU资源 |
| **收益极小** | 新增一天数据对训练好的模型参数影响微乎其微 |
| **过拟合风险** | 频繁用少量新数据更新，模型容易过拟合到最近的噪声 |
| **不必要** | 预测只需要输入窗口包含最新数据，参数不需要每日更新 |

---

## 关键注意事项

1. **禁止数据泄露**：预测时只能用**截至今日收盘**的数据，不能用未来数据训练
2. **窗口大小**：`lookback_window + predict_window ≤ max_context`（对于 Kronos-small/base 是 512）
3. **数据格式**：确保 `timestamps` 列能被 `pd.to_datetime()` 正确解析
4. **缺失数据**：volume 和 amount 如果没有，可以填 0，模型依然可以工作
